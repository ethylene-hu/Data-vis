特征维度说明

• 评论数：用户发表的评论总量

• 平均星级：用户给出的平均评分（1-5分）

• 互动性：用户获得的有用(useful)+有趣(funny)+酷(cool)评价总和

• 粉丝数：关注该用户的其他用户数量

• 精英年数：用户成为Yelp精英用户的年限

关键发现

• cluster 2（深蓝色）：互动性（136.56）和评论数（61.69）最高，代表“高互动活跃用户”（仅16个样本，极端特征群体）

• cluster 1：粉丝数和精英年数为0，可能是“孤立新用户”（仅1个样本，数据异常值）

• cluster 0：各项特征值中等（评论数23.17，互动性69.04），对应“普通评论者”（52999样本，占比2.67%）

• cluster 4：互动性（85.06）和粉丝数（1.51）中等，评论数25.45，为“中等互动用户”（976样本，占比0.05%）

最佳K值分析

optimal_k_analysis.png
最佳K值分析图，通过手肘法和轮廓系数法确定最优聚类数

算法验证与最佳聚类数确定

手肘法（左图）

• SSE（簇内平方和）随聚类数K增加而递减
• K=5处曲线斜率明显变缓，形成“手肘”拐点
• 表明K=5后继续增加聚类数，减少簇内误差的边际效益显著下降

轮廓系数法（右图）

• 轮廓系数衡量簇内紧凑性和簇间区分度
• K=2时系数最高（≈0.5），但簇划分过粗
• 综合业务需求（细分用户群体），最终选择K=5（平衡簇内紧凑性和业务可解释性）

数据异常提示

尽管算法确定K=5，但cluster 3的样本量占比97.28%，显著偏离正常聚类分布，建议：
• 检查数据预处理阶段的特征标准化是否正确
• 验证聚类算法初始化参数（如n_init）是否足够避免异常聚类
• 考虑使用层次聚类等其他算法交叉验证

收敛性意义

• 验证K-Means算法在该数据集上的稳定性
• cluster 3的中心在3次迭代后即趋稳定，其他聚类需更多迭代
• 建议增加n_init参数值（如n_init=50）以提升小聚类的稳定性

工程实现建议

针对极端样本量分布，建议：
• 采用加权K-Means算法，避免大聚类主导结果
• 先对数据进行过采样/欠采样，平衡聚类分布
• 使用DBSCAN等密度聚类算法作为补充分析

分析结论总结

用户群体特征显著分化

通过5大特征维度成功识别5类用户群体，其中cluster 3（97.28%）与其他群体存在显著特征差异，需特别关注数据分布合理性

聚类算法稳定性验证

手肘法与轮廓系数法支持K=5的合理性，但极端样本量分布（如cluster 3）可能影响小聚类的稳定性，建议补充其他算法验证

业务应用需考虑数据偏差

针对cluster 3的超大样本量，业务策略应分为两部分：维护核心少量活跃用户，重点唤醒大量低活跃用户
